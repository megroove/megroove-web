# CLAUDE.md — Megroove Web版 設計憲法

このファイルは Megroove **Webアプリ版** の設計判断と方針をまとめたものです。
実装の際は必ずこの内容を最優先の前提として扱ってください。判断に迷ったら、まず「最優先方針」に立ち返ること。

> 背景：iOS版（SwiftUI製）が App Store 公開のハードル（Apple Developer Program 登録の問題）に直面しているため、
> Web版を別途開発する。iOS版のコードは流用できないが、**設計思想・データ構造・画面設計・世界観はiOS版に準拠する**。
> iOS版はすでに多機能に育っているが、Web版は中核機能から段階的に作り、ブラウザで操作感を確かめながら拡張する。

---

## 1. コンセプト

日常的にコーヒーを淹れる行動を、詳細に記録・評価し、淹れ方の追求と再現を助けるWebアプリ。

- メインユーザー: こだわり派のコーヒー愛好家（自分用の趣味記録が出発点）
- 「美味しかったあの一杯を、もう一度淹れられる」ことを最終的なゴールにする

---

## 2. 最優先方針（この順で優先する）

competing する要求が出たら、必ずこの順位で判断する。

1. **記録の手軽さ** — 開いた瞬間に前回値で埋まっていて、変えたところだけ直せば保存できる。最小入力で完了できることが最重要。
2. **デザイン / 体験の心地よさ** — 数値を「入力する」より、つまみを動かす・チップを選ぶ感覚。比率の自動計算など「淹れている実感」を大事にする。
3. **再現性・分析の深さ** — ログから元の条件を再現できる。評価を定量化して後から分析できる。
4. **共有・コミュニティ** — 当面スコープ外。

> 重要: 「手軽さ」を犠牲にして高機能化しないこと。詳細機能は常に「畳んでおける／隠せる」前提で足す。

## 3. スコープ

### 実装済み（第1段階 完了）

- 記録（Brew の作成・編集）：最小入力モード ＋ 詳細入力 ＋ 写真撮影/保存
- 記録画面のカスタマイズ（ブロックのゾーン分け・並び順変更）
- カフェ来店記録（CafeVisit の作成・編集・詳細・写真）
- ライブラリ（記録一覧・フィルタ・再現・編集）＋ 3 表示モード（リスト / カード / レコード）
- 分析（好みのレーダーチャート＝星評価で加重平均、月間/年間トップ3ランキング）
- 豆・器具・レシピの管理（ストック）＋ 記録画面からのその場追加
- 保存完了アニメーション（波紋＋レコード盤フェードイン、節目演出）
- カフェイン管理（摂取量記録・残留量推定・就寝時の残留予測。表現は §12 の健康情報方針に従う）
- ホーム画面：今月のベスト記録・よく行くカフェランキング・推し豆・お気に入りアイテム枠
- 設定画面（/settings）：記録画面カスタマイズ + データ管理（エクスポート/インポート）
- ローカル完結のデータ管理（IndexedDB）＋ JSON エクスポート/インポート（カフェ記録含む・version 2）
- PWA 対応（ホーム画面追加・オフライン動作）
- 記録の信頼性（Phase A 完了）: 記録日時の編集（過去の一杯を記録可）・トースト通知＋削除アンドゥ・
  `navigator.storage.persist()`・バックアップリマインダー（ホーム/設定）・
  抽出ストップウォッチ（総抽出時間の計測・分秒入力）・豆の残量トラッキング（内容量から自動計算・飲み切りアーカイブ）
- 体験の心地よさ（Phase B 完了）: SVGラインアイコンシステム（絵文字全廃、`src/components/icons.tsx`）・
  ページ遷移フェード＋スケルトンローディング＋ボタン押下スケール（`prefers-reduced-motion` 対応）・
  ライブラリ検索（豆名/カフェ名/メモ/フレーバー）＋月別グルーピング・
  ホームの「今日」サマリ（本日の杯数/カフェイン残/連続記録日数）＋時間帯挨拶・
  safe-area 対応・数値入力の inputmode・ライトボックスのタップズーム
- 分析の深さ（Phase C 完了）: あなたのベスト条件（★4以上の記録が集中する比率/湯温/挽き目/抽出時間）・
  月別トレンドチャート（杯数＋平均評価、6/12ヶ月切替）・豆ごとの分析ページ（`/analysis/bean/:id`）・
  累計統計（総杯数/豆の消費量/カフェ支出/産地数）。分析ロジックは `src/components/analysis/stats.ts` に集約
- 産地入力のオートコンプリート: 国名＋代表的な地域名のマスター（`src/db/origins.ts`）とユーザー入力履歴から
  候補を提示（`src/components/OriginInput.tsx`、ひらがな→カタカナ正規化でマッチ）。
  豆フォーム（ストック/その場追加）とカフェ記録の3箇所で使用
- データ提供の土台（Phase D 完了 = docs/data-rewards-design.md の Phase 0）:
  匿名化エンジン（`src/provision/`、`megroove-provision/v1`）・仮名ID（HMAC-SHA256、`userSecret` は meta ストア＋
  バックアップに含めて機種変更対応）・設定→「データ提供の準備」でスコープ選択→プレビュー→JSONダウンロード
  （**送信機能はまだ無い**。写真/メモ/正確な時刻/豆名/カフェ名/器具名は常に除外）
- 後続フェーズの前倒し実装分: 蒸らしタイマー（プリセット＋カスタム秒設定・Wake Lock・対応環境で振動）・
  産地パスポート（`/passport`、産地別スタンプと杯数レベル）・初回起動フィーチャーツアー（`OnboardingTour`）・
  記録写真の拡大表示（`PhotoLightbox`、タップズーム）
- 健康情報の免責明文化（Phase E-1 完了）: カフェイン画面に免責・出典（食品安全委員会/EFSA）を明示、
  就寝時予測をユーザー目標基準の中立表現（目標内/目標超え）に変更、
  プライバシーポリシーに「健康に関する情報について」の節を追加（§12 の方針に基づく）

### 後続フェーズで追加（Phase E として合意済み・推奨順に1項目ずつ）

- 「1年前の今日の一杯」（ホームに過去の同日記録を表示）
- 「前回と同じ一杯」ワンタップ記録（星評価だけで保存完了）
- 針を落とす保存演出（トーンアーム追加。演出時間は延ばさない）
- 入力支援の横展開（精製方法チップ・品種/農園の履歴候補・フレーバー頻度順）
- カフェインの目安対比・「今飲むと就寝時◯mg」の事前提示
- シーン/飲み方チップ・デカフェ対応（任意・詳細ゾーン前提）

### 含まない（当面）

- データ提供モデル、サブスク課金（将来の収益機能。今は土台のみ意識）
- サーバー・ユーザーアカウント・ログイン（ローカル完結）
- 共有・SNS・コミュニティ

## 4. 技術方針（確定済み）

- **フレームワーク**: React 19 + TypeScript + Vite 8
- **スタイリング**: Tailwind CSS v4（`@tailwindcss/vite` プラグイン）
- **永続化**: IndexedDB via `idb` ライブラリ
  - DB名: `megroove`、バージョン: `3`
  - ストア: `beans` / `equipment` / `recipes` / `brews`（byBrewedAt インデックス）/ `cafeVisits`（byVisitedAt インデックス）/
    `meta`（内部値。データ提供の仮名ID導出に使う `userSecret` 等。`clearAllData` の対象外）
  - iOS版 SwiftData の「ローカル完結」と同じ思想。データは端末・ブラウザ内のみに保持
  - 各エンティティは **UUID** を持つ（iOS版と揃える）
- **ルーティング**: `react-router-dom` + `HashRouter`（GitHub Pages 対応。base: `/megroove-web/`）
- **公開**: GitHub Pages（`megroove` アカウント。サーバー費用ゼロ）
- **PWA**: `vite-plugin-pwa` + Workbox（generateSW モード）
- **バックアップ**: JSON ファイル書き出し／読み込み（`src/components/stock/DataTab.tsx`）

### localStorage キー一覧

| キー | 用途 |
|---|---|
| `megroove-settings` | AppSettings（就寝時間・カフェイン目標値） |
| `megroove-brew-layout` | 記録画面ブロックのゾーン分け設定 |
| `megroove-library-view` | ライブラリ表示モード（list/card/record） |
| `megroove-featured-bean-id` | ホーム画面の推し豆 ID |
| `megroove-featured-item` | ホーム画面のお気に入りアイテム（器具 or 写真） |
| `megroove-last-export` | 最終エクスポート日時（ISO）。バックアップリマインダーに使用 |
| `megroove-backup-snooze` | バックアップリマインダーの非表示期限（ISO） |

### Web特有の注意点（iOS版との違い）

- ハプティック（振動）・通知・タイマーのバックグラウンド動作には Web の制約がある。
  これらは「できる範囲で実装」。中核体験（記録・分析）を優先し、無理に作り込まない。
- 保存完了アニメーションのハプティックは、対応ブラウザでのみ Vibration API を使う（非対応環境では視覚演出のみ）。
- ブラウザのデータ消去で IndexedDB が消える可能性があるため、エクスポート機能で必ず補う。
- 写真は base64 JPEG（最大 800px）にリサイズして `photoDataUrl` フィールドに IndexedDB へ保存。
  `resizeImage(file, maxPx)` 関数は `src/db/helpers.ts` からエクスポート済み。

## 5. データモデル

概念モデルは「**淹れる = Bean × Recipe → Brew**」。
Brew（抽出ログ）が Bean と Recipe を参照し、そこから条件を再現できる。
各エンティティは UUID を持ち、IndexedDB のストアに保存する。

### Bean（豆）
- id (UUID), 名前 / 産地 / 農園(任意) / 品種(任意) / 精製方法(任意)
- 焙煎度（浅煎り〜深煎り）, 焙煎日(任意 → 経過日数 daysSinceRoast を計算して表示)
- 購入日 / 在庫メモ(任意)
- 内容量 initialAmountG(任意) — 記録の粉量合計から残量を**保存せず都度計算**（`calcBeanRemainingG`）
- 飲み切り finishedAt(任意) — 設定するとアーカイブ扱い（ピッカーの既定表示から除外、記録は保持）

### Equipment（器具）
- id (UUID), 名前 / タイプ / メーカー(任意) / サイズメモ(任意)

### Recipe（レシピ＝抽出テンプレート）
- id (UUID), 名前
- 既定の粉量 / 湯量 / 挽き目 / 湯温, 既定の器具(任意)
- ratio（= 湯量 / 粉量 を計算で導出）

### Brew（抽出ログ＝1回の記録）
- id (UUID), 日時 (brewedAt) — 既定は現在時刻だが、新規/編集どちらでも変更可能（過去の一杯をあとから記録できる）
- Bean 参照（UI上は必須だがデータ層は任意）, Recipe 参照(任意)
- その回の実値: 粉量 / 湯量 / 比率(計算) / 挽き目 / 湯温 / 器具 / 総抽出時間 / 注湯回数
- 評価（下記）, メモ, **photoDataUrl（base64 JPEG、任意）**
- cuppingAverage（カッピング5軸の平均。ランキングの第2ソートキー）
- caffeineAmount（粉量から自動推定。粉 1g ≒ 12mg）

### CafeVisit（カフェ来店記録）
- id (UUID), 訪問日時 (visitedAt), カフェ名 (cafeName)
- ドリンク名 / ドリンク種別 / サイズ / 豆の産地（いずれも任意）
- 評価（星 1〜5）, フレーバーチップ, カッピング（任意）
- 価格 / 推定カフェイン量（ドリンク種別×サイズテーブルから自動推定）
- **photoDataUrl（base64 JPEG、任意）**
- メモ

### 評価（Brew / CafeVisit に内包）
- 星評価（総合, 数値 1〜5）
- フレーバーチップ（文字列の配列, 複数選択）
- カッピングスライダー: 酸味 / 甘み / 苦味 / ボディ / 後味（各 任意, 1〜5 の 0.5 刻み）

### マスター生成の原則（重要）
Bean / Equipment / Recipe は **記録画面からもその場で生成できる** こと。
記録中に未登録に気づいても、記録画面を閉じずに追加でき、追加したものはマスターにも保存される（使い捨てにしない）。

## 6. 記録画面（このアプリの心臓部）

### 基本動作
- 「淹れる」を押すと、**前回のログ（または選んだレシピ）の値で初期表示** される。
- 何も変えなければそのまま保存できる。「変えたところだけ直す」体験。
- **最小入力で保存可**: 豆 + レシピ + 星評価 の 3 つだけで完了できる。

### 最小入力モード（開いた直後）
- 豆カード（豆名 / 焙煎度 / 焙煎からの経過日数を自動表示）
- レシピ行（「前回と同じ」。変更で別レシピや過去の名作ログを呼べる）
- 粉量 / 湯量 ＋ **比率を自動計算して表示**（例: 1:16）。値をいじると比率が即応答する
- 挽き目 / 湯温
- 評価: 星 + フレーバーチップ
- 「詳細を入力」の折りたたみ
- 「この一杯を記録する」保存ボタン

### 詳細を開いた状態（デフォルト）
- カッピングスライダー（酸味 / 甘み / 苦味 / ボディ / 後味）
- 器具（チップ選択）＋ 末尾に「**+ 追加**」（破線・アクセント色で「選ぶ」ではなく「作る」と区別）
- 抽出（総抽出時間 / 注湯回数）
- メモ
- 写真（📷ボタン → カメラ起動 or ギャラリー選択、800px にリサイズして保存）

### その場追加（器具・豆など）
- 「+ 追加」で記録画面を離れず、モーダル／シートで登録できる
- 入力は **名前 + タイプの最小限のみ**。ボタン文言は「**追加して、この記録で使う**」
- 登録後はそのまま選択済み状態で記録画面に戻る。マスターにも保存される
- 豆は焙煎日のみ実質必須、産地等は任意

### 保存完了アニメーション
- 通常保存（毎回）: 波紋が広がる中、レコード盤がフェードイン＋「一杯を記録しました」。約1.3秒。対応環境で軽いハプティック。
- 節目演出（1/10/30/50/100/200/365/500/1000杯目）: 紙吹雪＋「○杯目」拡大。タップでスキップ可能。
- レコード盤は同心円の溝＋中央レーベル（真上から見た液面）。アイコンと統一。

## 7. 記録画面のカスタマイズ（こだわり派向け）

記録画面を「ブロックの積み重ね」と捉え、各ブロックを 3 つのゾーンに振り分ける。

- ゾーン: **メインに表示 / 詳細にしまう / 非表示**
- デフォルト配置:
  - メイン: レシピ / 粉量・湯量 / 挽き目・湯温 / 評価（星） / フレーバー
  - 詳細: カッピング / 器具 / 抽出 / メモ / **写真**
- こだわり派は「カッピングスライダー」や「写真」を **メインに昇格** できる。
- 「豆」は **固定**（動かせない）。豆なしログは構造上ありえないため。
- カスタマイズは **表示順とゾーン分けのみ** 変更。データ構造は変えない。
- **非表示にしても、過去ログに記録済みの値は保持**（再表示で復活）。
- 設定は localStorage キー `megroove-brew-layout` に保存（`BrewLayoutSettings` 型）。
- 新規ブロック追加時は `loadBrewLayout()` が自動検出して detail ゾーンに追記（前方互換）。

### BrewBlockId 一覧
`recipe` / `dose_water` / `grind_temp` / `rating` / `flavors` / `cupping` / `equipment` / `extraction` / `note` / `photo`

| `/brew` | 記録（新規） | 上記 6・7 章のとおり |

| `/settings/privacy` | プライバシーポリシー | ローカル完結の方針・外部送信規律に基づく公表・健康情報の免責・将来のデータ提供の原則 |
| `/passport` | 産地パスポート | 記録した産地のスタンプ一覧（杯数によるレベル表示） |

### ライブラリ 表示モード（libary-view）
| モード | 表示形式 |
|---|---|
| `list` | 従来の1列リストカード |
| `card` | 写真トップの縦長（3:4）カード × 2列グリッド |
| `record` | アナログレコード盤（同心円の溝を写真全体に重ねる）× 2列グリッド |

写真なしの場合: カードモードは☕プレースホルダー、レコードモードは暗いビニール色 + コーラル色ラベル円。

### レーダーチャートとランキングの算出（iOS版に準拠）
- レーダー: カッピング5軸の **星評価による加重平均**（高評価の一杯ほど強く反映）。データが3杯以上のときのみ描画し、不足時は案内文の空状態を出す。下に「一言の気づき」を1文表示。母数は短く表示（例「42杯の平均」）。
- ランキング: ソートキーは 星評価 → cuppingAverage → 日時 の3段階。

### ホーム画面のランキング算出
- 対象: 当月のブリュー記録 + カフェ記録の両方

## 10. 開発の進め方

**いきなり全部を作らないこと。** 各段階の前に計画と確認事項を提示し、合意してから実装する。
Webでの操作感を都度ブラウザで確認しながら進める。

1. ✅ プロジェクト雛形（React）＋ IndexedDB のデータ層 ＋ 画面の骨組み
2. ✅ 記録画面（最小入力モード → 詳細 → その場追加 → 保存完了アニメーション）
3. ✅ 記録画面のカスタマイズ（ブロックのゾーン分け・並び順）
4. ✅ ライブラリ（一覧・フィルタ・再現・編集）＋ 3 表示モード
5. ✅ 分析（レーダーチャート＝加重平均・ランキング）
6. ✅ ストック管理、エクスポート/インポート、PWA対応
7. ✅ カフェ記録（CafeVisit）＋ カフェイン管理
8. ✅ ホーム画面強化（ランキング・推し豆・お気に入りアイテム）
9. ✅ 写真撮影・保存（ブリュー / カフェ両方）・ライブラリ表示モード3種
10. ✅ 蒸らしタイマー
11. ✅ 産地パスポート
12. 🔜 Phase E（1項目ずつ計画→合意→実装→ブラウザ確認のサイクルで進行中）

各段階でブラウザで動作確認できる状態を保つ。

## 12. 健康情報の扱い（確定方針）

カフェイン等、健康に関わる情報を扱うときは必ず以下を守る。

- 健康に関わる数値は「一般的なモデルによる推定・目安」として提示し、医学的助言・診断はしない。
- 「〜すべき」「危険」等の断定表現を使わない。「一般的な目安」「あなたの記録上の傾向」の提示にとどめる。
  例: 就寝時予測はユーザー自身が設定した目標値との比較（目標内/目標超え）で表現し、
  睡眠への影響そのものを断定するラベル（「影響あり」等）は使わない。
- 免責（医学的助言ではない・個人差がある・専門家への相談案内）をカフェイン画面と
  プライバシーポリシー「健康に関する情報について」に明示する。
- 参考値には出典（食品安全委員会・EFSA 等の公的機関）を添える。
- 睡眠等の外部ヘルスデータとの自動連携は行わない（ローカル完結の原則とも整合）。
